Introducció

El Typescript és Javascript amb llenguatge extra, hem de declarar variables amb el seu tipus, això ajuda a detectar errors durant el desenvolupament. No dóna avantatges de performance

Maneres de detectar errors

El Typescript té diverses maneres de detectar errors durant l'escriptura.

La primera manera seria creant plantilles d'objectes, així quan agafem un objecte al qual apliqui la plantilla, el Typescript ens avisarà de quins elements componen l'objecte i de quin tipus són. Les plantilles s'escriuen així: interface{propietat: tipus,...}

Les plantilles es creen amb la paraula interface, mira l'exemple:

interface User{ name: string; age: number; alive: boolean; }

Les interfaces poden ignorar propietats que no ens interessen quan per exemple, fem una petició axios per aobtenir l'objecte

Per a assignar la plantilla a l'objecte hem de posar "as plantilla" darrere de l'objecte en el moment que es declara, mira l'exemple:

const user1 = {name: "Roger", age:25, alive:true} as User

Si intentem definir una nova propietat per a un objecte creat amb una plantilla, donarà error

Un altre mètode per a que el TS ens ajudi a trobar errors, és definir el tipus dels paràmetres en les funcions. Es faria posanr dos punts i el tipus esperat a costat de cada paràmetre, mira l'exemple:

const funcio1 = (name: string, white: boolean)
